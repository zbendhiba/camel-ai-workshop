You will need the following tools installed on your machine to follow this workshop.

JBang

JBang lets you run Java scripts and install CLI tools without setting up a full project. It is the foundation for the Camel CLI used throughout this workshop.

Follow the official installation guide for your platform: JBang Installation Methods.

Once installed, verify that the jbang command is available:

Verify JBang
$ jbang version

Camel CLI

The Camel CLI provides the camel command used to run routes in every step of this workshop. Install it through JBang:

Install Camel CLI
$ jbang app install camel@apache/camel [jbang] Command installed: camel

Verify the installation:

Verify Camel
$ camel version

You can find more details about Camel CLI capabilities in the official documentation.

Ollama

Install Ollama from ollama.ai and pull the model used in this workshop:

Pull Model
$ ollama pull granite4:7b-a1b-h success

Note

You can also use one of these alternative models that we tested successfully: gemma4:e4b and ministral-3:8b.

Make sure Ollama is running before starting the workshop (ollama serve if it's not started automatically).

VS Code + Kaoto

Install VS Code and the Kaoto extension from the marketplace. This gives you a visual editor for .camel.yaml route files.

Alternative: OpenAI

If you prefer to use OpenAI instead of Ollama, you will need an API key. Set it as an environment variable:

Environment
$ export OPENAI_API_KEY=your-key-here

The OpenAI variant of each step uses application.properties where you can configure the model and API key.

Note

Ollama is also supported via its OpenAI-compatible API by setting camel.component.openai.baseUrl=http://localhost:11434/v1.
